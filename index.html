Políticas Corporativas

Política Corporativa de Seguridad de Datos

Código: DG-POL-PCSD-MINS-001 | Versión: 1.0

Este documento establece los principios fundamentales y los lineamientos generales para la protección de la información corporativa. Define los roles y responsabilidades en seguridad (como el Data Owner y el Data Steward) y establece estándares y controles para la clasificación, acceso y protección de datos. Además, detalla que los logs de auditoría deben conservarse por un mínimo de 5 años y estar protegidos contra modificaciones no autorizadas.

Política Corporativa de Calidad de Datos

Código: DG-POL-PCCD-MINS-001 | Versión: 2.0

Define los estándares, reglas y dimensiones de calidad para asegurar que los datos sean fiables, precisos y completos. Incluye un proceso de remediación de datos con etapas de identificación, análisis de causa raíz y corrección. Describe tipos de incumplimiento de la calidad de datos, clasificados como Leve (error puntual), Moderado (incumplimiento repetitivo) y Grave (impacto en operación o reputación).

Política Corporativa de Arquitectura y Metadata Analítica

Código: DG-POL-PCAMA-MINS-001 | Versión: 1.0

Establece los lineamientos para la arquitectura analítica, incluyendo la estructura de Datalake con el esquema Medallón (capas Bronce, Silver y Gold). Aborda el uso de Unity Catalog como catálogo centralizado y la gestión de metadatos (técnicos, de negocio y operacionales) para la trazabilidad y el linaje de datos.

Procedimientos

Procedimiento para Solicitar Accesos a Unity Catalog

Código: DG-PRO-PAUC | Versión: 1.0

Define el flujo estandarizado para la solicitud, evaluación y aprobación de accesos en Unity Catalog para proyectos de Analítica Avanzada. Detalla los roles y responsabilidades clave, como el Solicitante (quien requiere acceso) y el Analista de Diseño de Datos (quien asigna los permisos).

Procedimiento para Transferencia de Propiedad en Unity Catalog

Código: DG-PPRO-TPUC | Versión: 1.0

Establece las actividades necesarias para transferir la propiedad de tablas en Unity Catalog a un Service Principal. Aplica a las tablas creadas en los ambientes de desarrollo, calidad y producción, e involucra roles como el Gestor de TI y el Analista de Diseño de Datos para asegurar el control y la seguridad.

Procedimiento Corporativo de Remediación de Datos

Código: DG-PRO-PCRD-MINS-001 | Versión: 1.0

Define la secuencia de actividades para corregir incidencias de calidad de datos, desde la detección hasta el registro. Este procedimiento se aplica a datos críticos en sistemas como SAP, sistemas OT/PI, Data Lake y plataformas analíticas, con el fin de asegurar que los datos cumplan con las reglas de calidad y evitar problemas recurrentes.

Procedimiento para Configurar Metadata Driven

Código: DG-PRO-PCMD-MINS-001 | Versión: 1.0

Establece los pasos para configurar la arquitectura de metadatos de las iniciativas analíticas, permitiendo definir los parámetros de conexión con las fuentes de datos. Describe las responsabilidades de roles como el Gestor de TI y el Analista de Diseño del Dato en el llenado y validación de los formatos.
